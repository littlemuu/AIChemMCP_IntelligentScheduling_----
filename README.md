# AIChemMCP Intelligent Scheduling Project

## 项目简介

本项目围绕自动化实验室场景展开，目标是将 AIMCP 上层任务编排能力与底层智能调度算法结合起来，逐步形成一个能够支持实验任务生成、资源调度、执行推进和状态反馈的原型系统。

当前仓库保留了多个阶段和来源的代码与资料。为了避免阅读和使用时混淆，建议先阅读本说明，再选择对应目录。


## 仓库结构

当前仓库主要包含以下四部分内容。

(1) `documents/`

用于存放项目分析文档、算法说明文档、阶段性报告和相关资料图片。

(2) `lyz做的调度系统一些内容可供参考/`

这是一个独立的调度算法参考版本，主要用于理解调度模型、时间线机制和状态推进逻辑。

(3) `项目介绍和需求--最初版本/`

这是原始版本目录，保留了最开始拿到的 AIMCP 与调度代码结构，主要用于对照和回溯。

(4) `项目介绍和需求--调度集成版本/`

这是当前继续开发的主版本，包含 AIMCP 与智能调度算法的初步集成结果。


## 三个版本如何区分

本仓库中最容易混淆的是三个“版本”目录。下面按用途说明。

### 1. 调度参考版本

目录：

`lyz做的调度系统一些内容可供参考/`

定位：

(1) 这是调度算法参考代码。

(2) 主要价值在于帮助理解任务模型、资源模型、状态流转和时间线预留机制。

(3) 该目录更适合用于阅读和参考，不建议直接作为最终项目主目录继续开发。

适合什么场景：

(1) 想快速理解调度算法是怎么写的。

(2) 想查看 `Task / Workstation / Robot / Scheduler` 的基础实现方式。


### 2. 原始版本

目录：

`项目介绍和需求--最初版本/`

定位：

(1) 这是保留的原始版本。

(2) 主要用于保留项目最开始的目录结构和初始代码状态。

(3) 该目录建议仅用于对照、查阅和回退参考。

适合什么场景：

(1) 想比较“最初版本”和“集成版本”之间做了哪些变化。

(2) 想查看项目最早期的 AIMCP 和调度模块原始结构。


### 3. 调度集成版本

目录：

`项目介绍和需求--调度集成版本/`

定位：

(1) 这是当前主要开发版本。

(2) 该目录已经完成 AIMCP 与智能调度算法之间的初步连接。

(3) 后续如果要继续做功能扩展、调试、演示和汇报，建议统一在这个目录中进行。

当前已经具备的能力包括：

(1) `Agent -> ActionServer -> SchedulingRuntime -> Scheduler` 调用链已打通。

(2) 支持任务提交、状态查询、时间推进和运行至完成。

(3) 支持以下调试能力：

`robotic_reaction`

`robotic_measurement`

`robotic_characterization`

`scheduler_status`

`scheduler_advance`

`scheduler_run_until_complete`

## 运行方式

当前最推荐的调试入口位于：

`项目介绍和需求--调度集成版本/AIChemMCP-main/`

进入该目录后运行：

```powershell
python agent.py
```

该命令会启动一个本地演示流程，用于验证当前集成版本的最小闭环能力。

运行后可以观察到以下过程：

(1) Agent 启动 BO Server 和 Action Server。

(2) Agent 发现工具。

(3) Agent 提交反应任务和测量任务。

(4) Action Server 将任务转交调度运行时。

(5) 调度器推进任务直至完成。
